Unrestricted Resource Consumption.
When an API sets no limits on what any single client can use.
What "resources" means

In the context of APIs, resources aren't just data. They include anything the API consumes when processing a request:

📡
Network Bandwidth
Large responses, file downloads, media streams — each byte sent costs bandwidth.
⚙️
CPU & Memory
Complex queries, deep nested calls, large payload parsing — processing each request uses server resources.
💾
Database Connections & Queries
Every API call typically triggers a database query. Unlimited requests = unlimited DB load.
💸
Third-Party API Costs
SMS sends, email sends, AI inference, payment processing — each has a per-unit cost. Unlimited calls = unbounded bill.
⌛
Storage
File uploads, generated content, logs — unlimited storage consumption inflates cloud bills and fills disks.
The core problem

Without limits, a single malicious (or just careless) client can consume resources intended for thousands of users. The result is degraded service for everyone else, unexpected infrastructure costs, or complete outages.
